Users Management

With proper permissions, you can manage users and authentication methods.

On this page, you will find a table with :
  • An Add zone on the first line
  • the list of existing users with their informations on the next lines
Within each line, you could :
  • Delete a user when you click on
  • Modify a user when you click on
When adding or modifying, icons appears.
Enter or modify informations then click on to validate.
You could also click on to cancel, going back to initial values.
A user is defined by many fields :
  • his login
  • his password
  • his mail address
  • his role
  • authorization method
N.B. : admin user is not displayed and must be managed within application settings page.
N.B. : By security measures, password is never displayed

Authorization Methods

There are two methods : Local and External
  • Local : this account is used with authentication and authorization mechanism. Authentication will validate the password you enter here.
    Authorization will be used to apply role to the user and so his rights.
  • External : this account is used only with authorization mechanism. Password even if mandatory will never be used.
    Authentication will be against LDAP (only one available at the moment) and authorization will be against LDAP group membership and so role/rights associated.
    Account creation on this page adds an additional role to a specific user in addition of roles from his LDAP group membership.

Initial context choice

With external authentication, initial context of all roles/LDAP groups will be applied consecutively but only the last one will be taken under account
But if this user is also declared on this page, the initial context defined here will be finally applied.
