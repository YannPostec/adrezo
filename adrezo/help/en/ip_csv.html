CSV Import of IP addresses

It's possible for application administrators to import csv files in order to create/modify IP addresses in the database.
However, all temporay or migrating definitions are not available with this method.
Import file must contain less than 1000 lignes.

Each line must have the following fields (all mandatory unless specified) : KEYWORD,ID,CTX,SITE,SUBNET,IP,MASK,NAME,DEF,MAC

    Here an explanation of those fields :
  • KEYWORD : mandatory value = IP
  • ID : value for a creation = 0
  • CTX : Context name
  • SITE : Site Name
  • SUBNET : Subnet Name
  • IP : IP Address in special format (see next section)
  • MASK : Subnet mask in CIDR format (number between 2 and 32)
  • NAME : IP address name
  • DEF : IP address description
  • MAC : MAC address (non mandatory field)

IP Address Format

Every number of the IP address is represented by 3 digits with left completion with zeros if number is less than 3 digits.
Which gives for 10.178.76.3 => 010 178 076 003 => 010178076003
