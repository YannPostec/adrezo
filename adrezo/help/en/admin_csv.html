CSV Export and Import

CSV files Export

With proper permissions, you can export and import CSV files.

Only for the following tables :
  • Context
  • Site
  • Vlan
  • Subnet
  • IP

Export is done by clicking on

    NB : There is two different export for IP table :
  • Full Format : all the informations but no functionnal in import
  • Import Format : only the informations which could be treated in import

CSV files Import

To import, choose a text file in CSV format and click on
A progress bar appears as well as analysis comments.

    Bar progression informations
  • 25% : file upload
  • 15% : added/modified contexts
  • 15% : added/modified sites
  • 15% : added/modified vlans
  • 15% : added/modified subnets
  • 15% : added/modified IP addresses

When treatement is finished or when an error occurs, button appears to try again

Constraints on CSV files

Size file must be less than 3Mo.

    There is a maximum of lines the application could process :
  • Context : 5000 lines
  • Site : 2500 lines
  • Vlan : 2000 lines
  • Subnet : 1000 lines
  • IP : 1000 lines

Generalities about CSV files

Empty lines or lines beginning with other character then capitalize letter are ignored.
CSV import runs the same treatments as the web interface does, except for a light one with IP addresses.
IP addresses must be in database format, 12 digits, each part of IP address given a 3 digits block left completed by 0.
All fields are mandatory unless specified.

First field of each line determine the object type

  • Context = CTX
  • Site = SITE
  • Vlan = VLAN
  • Subnet = SUBNET
  • IP = IP

Second field is the database ID. If equals to 0, it will be a creation, if not a modification of the given ID.
Be careful that a modification of an unknown ID will do nothing without errors.

Files Prototype

Contexte : CTX,ID,NAME,SITE_MAIN
detail : CTX, id (0 or existing one), name, non used parameter (non mandatory)

Site : SITE,ID,CTX,COD,NAME
detail : SITE, id (O or existing one), context id, site code, name
Only changes of name or site code are taken into account when modifying records

Vlan : VLAN,ID,CTX,SITE,VID,DEF
detail : VLAN, id (0 or existing one), context id, site id, VLAN ID, description
Only changes of vlan id or description are taken into account when modifying records
Records with vlan id 0 are not alterable.

Subnet : SUBNET,ID,CTX,SITE,VLAN,IP,MASK,DEF,GW,BC
detail : SUBNET, id (0 or existing one), context id, site id, vlan database id, subnet IP, mask in CIDR format, description, gateway IP, broadcast IP
Only changes of description, subnet ip, mask, gateway or broadcast ip are taken into account when modifying records

IP : IP,ID,CTX,SITE,SUBNET,IP,MASK,NAME,DEF,MAC
detail : IP, id (0 or existing one), context id, site id, subnet id, IP address, mask in CIDR format, name, description, mac address (non obligatoire)
CSV Import can't deal with temporary IP or migrating IP.
So, only changes of name, description and mac address are taken into account when modifying records.
