Directories Management

With proper permissions, you can manage directories.
It is not possible to alterate the predefined local directory.

On this page, you will find a table with :
  • An Add zone on the first line
  • the list of existing directories list with their informations on the next lines
Within each line, you could :
  • Delete a user when you click on
  • Modify a user when you click on
When adding or modifying, icons appear.
Complete or modify informations then click on to validate.
You could also click on to canel, all fields return to their initial value.
A directory is defined by those fields :
  • A name
  • An order that determines the search order when users authenticates
  • Its type in a predefined list :
    • Local : User local database. It already exists and cannot be modified or deleted.
    • LDAP : Only external type available today. Choose this type to query a LDAP external directory (like OpenLDAP or MS Active Directory)
    • Other external types like Radius or TACACS could be added to this list in future versions.
  • the configuration for external directories

LDAP Configuration

LDAP Configuration is not available at directory creation. You configure it once created
The configuration window contains multiple fields, mandatory ones are marked in bold.
Others fields would have an default value if left empty.
An explanatory text in italic is present below each field, read them for explanations.

N.B. : By security measures, password is never displayed
