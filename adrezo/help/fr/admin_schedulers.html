Gestion des Tâches Planifiées

Avec les droits nécessaires, vous pouvez agir sur les tâches planifiées pré-programmées de l'application.

Vous trouverez dans la page une liste des tâches (job) avec :
  • son ID
  • si la tâche doit être activée
  • son action
  • la possibilité d'exécuter manuellement la tâche
  • des paramètres additionnels pour certaines tâches
  • des statistiques sur des éléments se rapportant à la tâche
  • le nom du job pour le retrouver dans la liste des tâches
Après modification d'un des paramètres, cliquer sur pour valider.
Vous trouverez également en début de page le moyen d'afficher la liste des tâches planifiées et leur prochaine exécution.
Cette liste s'affiche en cliquant sur
Certaines tâches peuvent être exécutés manuellement en cliquant sur . Consultez ensuite les logs pour suivre l'exécution.

Détails des jobs :

  • Job 1 : A chaque connexion d'un utilisateur, la date est enregistrée dans une table avec ses préférences
    Cette table est nettoyée des utilisateurs qui ne se sont pas connectées depuis x jours
  • Job 2 : Chaque modification de stock est enregistrée
    La table est nettoyée des lignes de mouvement vieilles de x jours
  • Job 3 : Lorsque des photos sont supprimes dans le module photos, elles sont juste supprimées de la base de données. Les fichiers physiques restent.
    Ce job permet de comparer les fichiers physiques avec les informations de la base et de supprimer les fichiers sans référence.
    On peut lancer manuellement cette vérification en cliquant sur le bouton
  • Job 4 : Les adresses IP temporaires sont valides jusqu'a une certaine date. Cette tâche permet d'envoyer un mail aux utilisateurs dont la date de ces IP est dépassée.
    Si l'utilisateur n'existe plus, les informations sont envoyées au compte admin
  • Job 5 : Les adresses IP en migration sont valides jusqu'a une certaine date. Cette tâche permet d'envoyer un mail aux utilisateurs dont la date de ces IP est dépassée.
    Si l'utilisateur n'existe plus, les informations sont envoyées au compte admin
  • Job 6 : Les subnets qui ne sont pas dans la norme sont mis automatiquement dans le surnet correspondant si ce dernier existe.
  • Job 7 : Recuperation de la liste des equipements dans la base de données Cacti et injection dans celle d'Adrezo. Lancé quotidiennement
  • Job 8 : Recuperation des statistiques de disponibilité réseau depuis la base de données Cacti, injection dans celle d'Adrezo et remise à zéro des données dans Cacti. Récurrence horaire,quotidienne ou mensuelle.
  • Job 9 : Aggregation des statistiques horaires vers la moyenne journalière
  • Job 10 : Aggregation des statistiques journalières vers la moyenne mensuelle
  • Job 11 : Collecte des informations DHCP de chaque serveur déclaré

Les planifications des tâches sont gérées dans le fichier quartz-config.xml présents dans WEB-INF/classes
La gestion des logs est gérée dans le fichier log4j.xml au même endroit.
