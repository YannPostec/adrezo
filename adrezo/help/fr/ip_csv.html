Import d'adresses IP au format CSV

Il est possible pour les administrateurs de l'application d'importer des fichiers CSV afin de créer/modifier les adresses IP contenus dans la base de données.
Il n'est pas possible d'intégrer toutes les notions d'IP temporaire ou de migration avec cette méthode.
Le fichier d'import ne doit pas contenir plus de 1000 lignes.

Chaque ligne du fichier CSV devra être constituée des champs suivants (tous obligatoires sauf si précisé) : KEYWORD,ID,CTX,SITE,SUBNET,IP,MASK,NAME,DEF,MAC

    Voici une explicaton de ces champs :
  • KEYWORD : valeur obligatoire : IP
  • ID : valeur pour une création : 0
  • CTX : Nom du contexte
  • SITE : Nom du site
  • SUBNET : Nom du sous-réseau
  • IP : Adresse IP dans un format particulier (voir section suivante)
  • MASK : Masque de sous-réseau au format CIDR (nombre de 2 à 32)
  • NAME : Nom pour cette adresse IP
  • DEF : Libellé pour cette adresse IP
  • MAC : MAC de cette adresse IP (champ non obligatoire)

Format de l'adresse IP

Chaque nombre de l'adresse IP est représenté par 3 chiffres avec une complétion par des zéros à gauche si le nombre ne contient pas 3 chiffres.
Ce qui donne pour 10.178.76.3 => 010 178 076 003 => 010178076003
