Gestion des subnets dans les modèles de sites

Avec les droits nécessaires, vous pouvez gérer les subnets des modèles de sites.

Vous trouverez dans la page deux zones :
  • une zone d'ajout
  • la liste des subnets existants avec leurs informations
Pour chaque ligne de la liste, vous pouvez :
  • Effacer un subnet en cliquant sur
  • Modifier un subnet en cliquant sur
En cas d'ajout, entrez les différentes informations puis cliquez sur pour valider.
En cas de modification, modifiez les différentes informations puis cliquez sur pour valider.
Dans les deux cas, vous pouvez cliquer sur pour annuler et revenir aux valeurs initiales.

Il est possible de n'afficher dans le tableau que les lignes contenant la chaîne de caractères rentrée dans le champ Rechercher

Un subnet est défini par plusieurs champs :
  • son nom
  • son ip de subnet
  • son masque
  • son éventuelle passerelle
  • son vlan
  • son ip de broadcast, calculé automatiquement
  • sa liste de surnet pour un site non-standard
A noter : #CODE# sera remplacé par le code du site, #SITE# par le nom du site

Cas des subnets pour modèle standard

Il suffit de rentrer le masque pour que tous les choix possibles d'ip de subnet soient calculés.
Ce sont les subnets possibles dans le surnet du site, calculé sur une base ip de 0.0.0.0
Attention les ip des subnets existants ne sont pas retirés de la liste proposée

Cas des subnets pour modèle non-standard

Le subnet sera le premier disponible dans la liste de surnets définis, sa valeur est donc toujours 0.0.0.0
