Gestion des API

Avec les droits nécessaires, vous pouvez restreindre la liste de subnets autorisés pour les utilisateurs ayant accès aux API.

Vous trouverez dans la page un tableau avec :
  • une zone d'ajout en première ligne
  • la liste des utilisateurs existants avec leurs informations sur les lignes suivantes
Pour chaque ligne de la liste, vous pouvez :
  • Effacer un utilisateur en cliquant sur
  • Modifier un utilisateur en cliquant sur
En cas de modification, les informations sont reportées dans la zone d'ajout.
En cas de changement d'utilisateur, la zone repasse en mode ajout.
Entrez ou modifiez les différentes informations puis cliquez sur pour valider.
Vous pouvez également cliquer sur pour annuler, tous les champs reviennent à leur valeur initiale.

N.B. : L'utilisateur n'est disponible dans la liste que s'il possède le droit API sur au moins un des contextes dans la gestion du role qui lui est attribué

Liste des Subnets autorisés via API

Par défaut, l'utilisateur a accès à l'ensemble des subnets, vous pouvez restreindre à certains subnets uniquement.
Les contextes présentés dans la liste sont ceux possédant le rôle API dans le rôle attribué à l'utilisateur.
Pour modifier cette liste, utiliser l'icône pour ouvrir une fenêtre :
Dans la partie droite de cette fenetre se trouve la liste des subnets autorisés.
Dans la partie gauche se trouvent les subnets que vous pouvez autoriser.
La partie gauche est remplie à chaque fois que vous choisissez un contexte, vous pouvez affinez en sélectionnant un site.
Il est possible de bouger un ou plusieurs éléments entre les listes en utilisant les icônes :
  • Passer de la colonne de gauche à celle de droite
  • Passer de la colonne de droite à celle de gauche
Il vous suffit de valider en cliquant sur pour remplir le champ "Liste des Subnets" non modifiable sinon.
